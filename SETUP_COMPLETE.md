## How It Works

```
Your Code (main.fsh)
        ↓
    Rust Compiler (fluxc)
    Generates x86-64 Assembly
        ↓
    main.asm (auto-created)
        ↓
    GNU as Assembler
    Creates machine code
        ↓
    main.o (auto-created)
        ↓
    GNU ld Linker
    Creates executable
        ↓
    program (auto-created)
        ↓
    ./program ← RUN THIS
```

## Key Facts

✅ **Automatic Code Generation**
- main.fsh → main.asm (by fluxc compiler)
- main.asm → main.o (by GNU as)

## Build Commands

| Command | Does | Output |
|---------|------|--------|
| `make` | All steps + run | Executable runs |
| `make build` | Compile + assemble + link | program created |
| `make compile-asm` | .fsh → .asm | main.asm created |
| `make compile-binary` | .asm → executable | program created |
| `make run` | Execute | Program runs |
| `make clean` | Remove generated files | Clean state |
| `make distclean` | Clean all + compiler | Full reset |
| `make help` | Show all commands | Help text |

## What's Generated?

After `make build`:

### `main.asm`
- x86-64 assembly code
- AT&T syntax (GNU as compatible)
- Generated by fluxc compiler
- Safe to ignore

### `main.o`
- Binary object file
- Machine code in ELF format
- Generated by GNU as
- Safe to ignore

### `program`
- The actual executable
- Can run with `./program`
- Generated by GNU ld
- This is what you run

## Troubleshooting

| Problem | Fix |
|---------|-----|
| `fluxc: not found` | `cd flux_compiler && cargo build --release` |
| `as: not found` | `sudo apt-get install binutils` |
| Syntax errors | Check main.fsh syntax (read LANGUAGE_GUIDE.md) |
| No output | Make sure main.fsh calls `print()` |
| Won't compile | Read error message - it tells you exactly what's wrong |
